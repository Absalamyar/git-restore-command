راهنمای دستور git restore

آموزش کاربردی لغو تغییرات در پروژه با مثال‌های HTML

۱. لغو تغییرات در یک فایل (قبل از مرحله Add)

اگر کدی را اشتباهاً در فایل نوشتید و می‌خواهید فایل به آخرین وضعیت کامیت شده برگردد:

# بازگرداندن فایل به آخرین کامیت
git restore index.html

مثال: فرض کنید خط زیر را اشتباهاً اضافه کرده بودید، با دستور بالا این خط حذف می‌شود:

<h1>این یک متن اشتباه است!</h1>

۲. خارج کردن فایل از صف کامیت (بعد از مرحله Add)

اگر دستور git add را اجرا کرده‌اید اما پشیمان شده‌اید و نمی‌خواهید این فایل کامیت شود:

# خارج کردن فایل از حالت Staged (کدهای شما حذف نمی‌شوند)
git restore --staged index.html

نتیجه: فایل شما از حالت آماده‌باش برای کامیت خارج می‌شود تا بتوانید تغییرات بیشتری روی آن اعمال کنید.

۳. لغو تمام تغییرات پروژه به صورت یکجا

اگر تغییرات زیادی در چندین فایل HTML و CSS انجام داده‌اید و می‌خواهید همه چیز به حالت اول برگردد:

# هشدار: این دستور تمام کدهای کامیت نشده را برای همیشه پاک میکند!
git restore .
